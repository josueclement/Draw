# Release 1.2.0 — version bump, changelog, documentation accuracy

## Objective

Cut the third release of Draw: stamp the build `1.2.0`, document the work merged since `1.1.0` in the
changelog and roadmap, and keep the public-facing docs in line with reality. No build/distribution
machinery is added here — that remains the separately-tracked public-release blocker.

## Context

`main` carries two changes on top of the `1.1.0` tag (`3c220b5`), both backward-compatible refinements
of existing connector commands:

- `efbbe4e` — **duplicate a shape's connectors when duplicating it**: `Ctrl+D` now also clones any
  connector with *either* end touching the selection (was both-ends-only), resolving endpoints via
  `CloneArranger`. Copy/paste behaviour is unchanged.
- `631392f` — **anti-crossing connection spacing**: the *Space connections* action orders each side by
  the far shape's position so fanned-out connectors stop crossing.

New behaviour with no breaking changes means a **MINOR** bump under Semantic Versioning, so the release
is **1.2.0**. No README/roadmap drift this time beyond documenting the two features (the duplicate-
connectors change was the only one not yet in the roadmap; the anti-crossing change was already
documented under *Connection point spacing*).

## Decisions

- **Version:** `1.2.0` in `Directory.Build.props` (single source of truth, solution-wide) and a
  matching `1.2.0.0` in `src/Draw.App/app.manifest` (Win32 SxS identity; cosmetic for .NET but kept in
  sync). The in-app label needs no change — `ShortcutHelpViewModel.VersionLabel` reads the assembly.
- **Scope:** documentation + metadata only. No publish script, no CI/CD, no packaging — source + tag
  only, as with `1.0.0` and `1.1.0`.
- **Changelog category:** both features are refinements of existing commands, so they go under
  **Changed** (not Added).
- **Visual verification:** the two connector features are confirmed working by the owner; described as
  shipped with no "pending visual verification" caveat. WSL2 cannot render the GUI, so the Shift+H
  version check below is the only owner-side manual step left.
- **Conventions:** follows the project scheme (`documentation/roadmap.md` + dated
  `documentation/plans/*.md`), mirroring `2026-06-30-release-1.1.0.md`.
