# Release 1.0.0 — version stamp, in-app version, LICENSE, CHANGELOG, crash handler

## Objective

Prepare the first public release of Draw: give the build an explicit `1.0.0` version, surface it in
the app, and add the minimum public-release artifacts that were missing (license, changelog, a crash
safety net). Distribution/CI — the other public-release blocker — is deliberately deferred and
tracked separately in the roadmap.

## Context

The build had **no explicit version**: it relied on MSBuild's implicit `1.0.0.0` default, and the
version was not shown anywhere in the UI. For a public release the repo also lacked a `LICENSE`, a
`CHANGELOG`, and any crash safety net (an unhandled exception closed the app silently with no record).

## Decisions

- **Version config:** `Directory.Build.props`, solution-wide, version **plus** product metadata.
  Single source of truth — the libraries ship with the app, so they share its version. Stamping the
  test assemblies too is harmless.
- **In-app display:** top-right of the Shift+H help overlay header, text `Draw 1.0.0`, read from the
  assembly (no duplicated literal).
- **License:** MIT, `Copyright (c) 2026 Josué Clément`.
- **Crash handler:** log every unhandled exception to `%APPDATA%/Draw/logs/`; show a dialog for
  UI-thread crashes, then exit. Log-only for crashes we cannot surface a dialog for.
- **Distribution/CI:** out of scope here; logged as a tracked follow-up in the roadmap.

## Changes

| Area | File(s) |
|------|---------|
| Version + metadata | `Directory.Build.props` (new product-identity `PropertyGroup`) |
| In-app version | `src/Draw.App/ViewModels/ShortcutHelpViewModel.cs` (`VersionLabel`), `src/Draw.App/Views/ShortcutHelpView.axaml` (header grid) |
| Crash handler | `src/Draw.App/Services/CrashHandler.cs` (new), `src/Draw.App/Program.cs` (`CrashHandler.Register()`) |
| License | `LICENSE` (new), `README.md` (License section) |
| Changelog | `CHANGELOG.md` (new) |

Notes:
- `src/Draw.App/app.manifest` already declared `version="1.0.0.0"`, so it needed no change.
- `<Version>1.0.0</Version>` yields AssemblyVersion/FileVersion `1.0.0.0` and InformationalVersion
  `1.0.0+<sha>`; `VersionLabel` uses `GetName().Version.ToString(3)` → clean `1.0.0`.
- The crash handler reuses the `%APPDATA%/Draw/` path convention from `RecentFilesService`. The dialog
  hooks `Dispatcher.UIThread.UnhandledException` (Avalonia 12, WPF-style) so it can render while the
  UI thread is still alive; `AppDomain.UnhandledException` + `TaskScheduler.UnobservedTaskException`
  cover the log-only paths.

## Verification

- `dotnet build Draw.slnx` — clean (nullable + analyzer warnings are errors).
- `dotnet test --solution Draw.slnx` — existing suite still green (no logic touched).
- **On Windows (cannot render on WSL2):** run the app, press **Shift+H**, confirm `Draw 1.0.0`
  appears top-right of the help card; trigger an unhandled exception and confirm a log file lands in
  `%APPDATA%/Draw/logs/` and the error dialog appears before exit.
- Optional: confirm `Draw.App.dll` FileVersion/ProductVersion read `1.0.0` / `1.0.0+<sha>`.
