# JSON-configured keyboard shortcuts (vim/blender-style chords): Plan

Branch: `feature/json-chord-keymap`.

Keyboard shortcuts were hard-coded in two places — a `<Window.KeyBindings>` block in
`MainWindow.axaml` and a `switch` in `DiagramView.axaml.cs` `OnKeyDown`. This replaces both with a
single **data-driven keymap**: one window-level dispatcher reads a merged keymap (baked-in defaults +
optional user override JSON) and handles **single gestures** (`Ctrl+S`) and **multi-key chords**
(`a s r` → arm the rectangle tool, `a c a` → arm the association connector). Everything is remappable;
nothing stays hard-coded.

## Behaviour

- **Arm-the-tool semantics**: an "add" chord arms the matching tool (then click/drag on the canvas to
  place — connectors need two endpoints, so this is uniform across all node/connector kinds). Non-add
  actions (delete, copy, undo, align, zoom…) invoke immediately.
- **Chord matching**: case-insensitive; bare letters/digits ignore Shift but require no Ctrl/Alt/Meta
  (so `Ctrl+C` and a bare `c` never collide). A complete binding that is also the prefix of a longer
  one waits (timeout commits the shorter); a key that breaks the prefix commits the shorter then
  reprocesses. Escape cancels an in-progress chord; an unmatched sequence flashes "no binding".
- **Feedback**: the pending chord (e.g. `a s`) and transient messages show in the bottom status bar; a
  ~1 s idle timeout discards the buffer.
- **Defaults + override**: built-in defaults are baked in; the user overrides/extends via
  `%APPDATA%/Draw/keymap.json` (merged per-key-sequence; empty `action` unbinds). A commented
  `keymap.example.json` is written there on first run. Missing/corrupt user file → defaults (graceful,
  mirroring `RecentFilesService`).
- **Untouched**: in-place text editing keeps its keys — the dispatcher is suppressed whenever a
  text-entry surface (TextBox / AutoCompleteBox / ComboBox) has focus; Escape-exits-edit and the
  member/column editing keys are unchanged.

## Components (new — `src/Draw.App/Input/`)

1. **`KeymapModels.cs`** — `KeymapBinding(Keys, Action)`, `KeymapFile(Bindings)` JSON DTOs.
2. **`KeyChord.cs`** — `KeyStroke(Key, Modifiers)`, `ParsedBinding(Strokes, Action)`, and
   `KeyStrokeSequenceComparer` (order-sensitive structural equality for dictionary/set keys).
3. **`KeyGestureParser.cs`** — parses `"Ctrl+Shift+S"` / `"a s r"` → `KeyStroke[]` (one path via
   `Enum.TryParse<Key>`, digits mapped to `D0..D9`); `Describe(...)` renders strokes for the status bar.
4. **`DefaultKeymap.cs`** — baked-in default JSON (`const string`) + the commented example JSONC.
5. **`KeymapService.cs` / `IKeymapService`** — loads defaults, merges the user file, writes the example,
   exposes parsed `Bindings`. APPDATA path + graceful-fallback exactly like `RecentFilesService`.
6. **`KeymapStatusViewModel.cs`** — `Pending` / `TransientMessage` / computed `Display` for the status bar.
7. **`KeymapActionRegistry.cs`** — maps action ids → command resolvers, evaluated **lazily** against the
   live `ShellViewModel`/`ActiveDocument` (document-scoped commands always target the current tab). The
   parameterized families (`tool.shape.*`, `align.*`, `zorder.*`…) are generated by looping the enums.
8. **`ChordInputDispatcher.cs`** — the buffer + timeout state machine; precomputes an exact-match
   dictionary and a prefix set; `bool HandleKeyDown(KeyEventArgs)` / `void Reset()`.
9. **`Configuration/KeymapOptions.cs`** — `ChordTimeoutMs` (1000), `TransientMessageMs` (1200).

## Integration (edits)

- **`ServiceCollectionExtensions.cs`** — `Configure<KeymapOptions>` + singletons `KeymapStatusViewModel`,
  `IKeymapService`, `KeymapActionRegistry`, `ChordInputDispatcher`.
- **`ShellViewModel.cs`** — inject + expose `KeymapStatus`.
- **`MainWindow.axaml.cs`** — inject `ChordInputDispatcher`; `AddHandler(KeyDownEvent, …, Tunnel)` so
  plain letters reach the dispatcher before a focused control; focus-based suppression guard; `Reset()`
  on `Deactivated`.
- **`MainWindow.axaml`** — delete the `<Window.KeyBindings>` block and the menu `InputGesture` hints; add
  a status-bar `TextBlock` bound to `KeymapStatus.Display`.
- **`DiagramView.axaml.cs`** — remove the clipboard/Delete block from `OnKeyDown`; keep the TextBox guard
  and Escape→`EndEditing` fallback.

## Default scheme (excerpt)

Singles: migrated `Ctrl+N/O/S`, `Ctrl+Shift+S`, `Ctrl+W`, `Ctrl+Z/Y`, `Ctrl+C/X/V/D`, `Delete`,
`Ctrl+Shift+F`, `Ctrl+Shift+{L,C,R,T,M,B}` align, `Ctrl+Shift+{H,V}` distribute,
`Ctrl+(Shift+)Oem[Open/Close]Brackets` z-order, `Escape` → `tool.select`.
Chords (prefix `a` = add): shapes `a s {r,o,e,c,d,p,z,t,n}`, connectors `a c {a,d,g,o,n,r,p,i,e,l}`,
class `a u {c,i,e}`, use-case `a k {a,u,b}`, ER table `a t`; convenience `z {i,o,r}`, `g f`, `t t`,
`x {i,s,c}`.

## Update — category tool menus (replaces per-shape chords)

One chord per shape/connector proved excessive. The granular `tool.*` chords are removed from the
defaults and replaced by **two keyboard-opened, category-grouped context menus**:

- `Shift+S` → **Shapes** menu, `Shift+C` → **Connectors** menu (both remappable; new actions
  `menu.shapes` / `menu.connectors`). Each is grouped into **Standard / UML / Use case / ER** submenus.
- Picking an item **arms the matching tool** (unchanged placement flow). Items show the ribbon icons and
  carry `_`-mnemonic **access keys**; the menu opens at the cursor on the active canvas.
- All granular `tool.*` action ids remain **bindable** in `keymap.json` for power users.

Design: a `menu.*` action runs `ShellViewModel.ShowToolMenuCommand`, which raises `ToolMenuRequested`
(same event→code-behind idiom as the export commands); `MainWindow` opens the matching XAML-declared
`ContextMenu` (in `Window.Resources`) on the active `DiagramView` via `OpenToolMenu` (reusing the
arrange-menu `Dispatcher.UIThread.Post(() => menu.Open(Viewport))` deferral). The menus are fully static
XAML (icons + access-key headers + `x:Static` `CommandParameter`s); the shared arm command is assigned per
item in code-behind (`WireToolMenu`, by `CommandParameter` type) — mirroring `WireToolDropdowns`.

**Shift is now significant** for letter gestures (the dispatcher's bare-letter Shift-stripping was
removed) so `Shift+S`/`Shift+C` are distinct from `s`/`c`. Files: `ViewModels/ToolMenuFamily.cs` (new),
`ShellViewModel`, `Input/KeymapActionRegistry`, `Input/ChordInputDispatcher` (`Normalize`),
`Input/DefaultKeymap`, `Views/MainWindow.axaml(.cs)`, `Views/DiagramView.axaml.cs`. Build clean; pending
visual verification on Windows.
