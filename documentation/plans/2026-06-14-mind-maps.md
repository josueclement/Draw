# Mind maps — topics, hover-`+` child creation, tapered branches — Implementation plan

Branch: `phase06-mind-maps`. Phase 6 (a new diagram type).

Draw could place ~29 shape kinds and link them, but had no **mind-map** workflow. This phase adds one:
drop a topic, then grow the tree by hovering a topic and clicking a `+` on any of its four sides to
spawn a linked child — with **organic, tapered** links that are thick near the central topic and thin
toward the leaves.

`Rectangle`/`RoundedRectangle` already existed, so the *shapes* aren't new. What's new is a **mind-map
node identity** that carries the affordance, the **hover-`+`-to-spawn-child** interaction, and
**depth-scaled, along-link tapered connectors**.

## Decisions

- **Dedicated mind-map kinds** — two new `ShapeKind`s (`MindMapTopic`, `MindMapTopicRounded`) rendered
  exactly as plain / rounded rectangles. Only these show the `+` buttons; ordinary rectangles are
  untouched.
- **Taper = depth-scaled + along-link** — the full "tree-branch" look: a branch interpolates from
  `WidthAt(parentDepth)` at its source to `WidthAt(parentDepth+1)` at its child, and `WidthAt` shrinks
  geometrically with depth, so width flows continuously from trunk to twig.
- **Child placement = offset + collision nudge** — placed a gap out on the clicked side, then fanned
  along that side until it clears existing nodes. No global auto-layout; the user drags freely after.
- Child **inherits the parent's shape kind & style**, **auto-enters inline editing**, and links carry
  **no arrowheads**.
- **New `DiagramType.MindMap`** + a *New Mind Map* command that pre-seeds one central topic.
- A connector tapers **only when created via `+`** (an explicit, serialized `Connector.IsMindMapBranch`
  flag) — manually drawn links stay normal, even between two topics.

## What changed

### Model (`Draw.Model`) — additive, back-compatible
- `Nodes/ShapeKind.cs`: appended `MindMapTopic = 29`, `MindMapTopicRounded = 30`.
- `Connectors/Connector.cs`: added `bool IsMindMapBranch` (default false; carried in `Clone()`).
- `Documents/DiagramType.cs`: appended `MindMap = 4`.

### Pure logic (`Draw.Diagramming/MindMap`) — headless-tested
- `MindMapHierarchy.ComputeDepths(connectors)` — BFS depth-from-root over branch connectors
  (source→target); robust to cycles / missing roots.
- `MindMapBranchStyle.WidthAt(depth)` — `max(MinWidth, BaseWidth · Falloff^depth)`; the single source
  of branch widths (base 9 / falloff 0.68 / floor 2 px), shared by canvas + SVG.
- `TaperedStroke.BuildOutline(centerline, startWidth, endWidth)` — closed variable-width ribbon
  polygon (each sample offset ±half-width along its normal; width interpolated by arc length).
- `Geometry/ShapeOutline.cs`: the two kinds route as plain rectangles.

### Rendering (`Draw.App`)
- `ShapeGeometryBuilder` / `ShapeSvgPathBuilder`: the two kinds map to the rectangle / rounded-rect
  geometry.
- `ConnectorViewModel`: `IsMindMapBranch`, `SetBranchDepth(parentDepth)`, `GetBranchOutline()`, and a
  filled `BranchGeometry`; decorations are suppressed for branches; a ribbon-outline selection halo.
- `DiagramView.axaml` connectors layer: a filled `BranchGeometry` path (shown for branches) replaces
  the stroked line; `DiagramSvgRenderer.EmitConnector` emits the same filled polygon for export parity
  (PNG already captures the live `Path`).

### Interaction / UX (`Draw.App`)
- `ShapeNodeViewModel`: `IsMindMap`, `ShowChildButtons` (hidden while editing).
- `DiagramView.axaml` shape template: four hover `+` buttons (one per side) using the existing
  `:pointerover` idiom, gated on `ShowChildButtons`.
- `DiagramView.axaml.cs` `OnCreateChildClick`: reads the side from the button `Tag`, calls
  `CreateChildNode`, then opens the child for typing (reusing `FocusEditorFor`).
- `DiagramDocumentViewModel`:
  - `PlaceNode`/`AddConnector` refactored into CaptureUndo-free cores (`PlaceNodeCore`,
    `LinkNodesCore`) so a child + its branch are **one undo step**.
  - `CreateChildNode(parent, side)`: inherits kind/style, side-aware placement + collision nudge,
    side-centre branch anchors, selects the child.
  - `RefreshMindMapBranches()` recomputes each branch's source depth; called at the end of
    `RebuildConnectors()` (load/undo/redo/delete/paste) and after each child add. Node *moves* need no
    refresh — the branch re-routes and `BranchGeometry` reads the live route.

### Palette & diagram type (`Draw.App`)
- `ToolboxViewModel`: two tools ("Topic", "Rounded topic") + a `MindMap` shape category and
  `MindMapHeader`.
- `MainWindow.axaml`: a **Mind map** ribbon group (Insert) with the two-tool dropdown; *New Mind Map* in
  the File menu and the Home File ribbon group. `MainWindow.axaml.cs` wires the dropdown to the
  shape-arm command.
- `ToolMenus.axaml`: a **Mind map** submenu in the Shift+S tool menu.
- `ShellViewModel.NewMindMapCommand` → `CreateNew(DiagramType.MindMap)`; the factory seeds one
  `MindMapTopicRounded` "Central topic" near the initial viewport.

## Tests
- `MindMapBranchStyleTests`, `MindMapHierarchyTests`, `TaperedStrokeTests`, and two `ShapeBoundary`
  cases for the new kinds. Full suite green (363).

## Verification (visual — must run on Windows/macOS; WSL2 has no display)
*New Mind Map* opens with a central topic → hover shows four `+` → clicking each side spawns a
connected child on that side that opens for typing → links render as smooth filled branches, thick at
the root and tapering with depth and toward each child, no arrowheads → moving a node re-routes the
branch and keeps the taper → deleting a node prunes its branch and re-thins descendants → undo/redo
treats each child-add as one step → save/reopen round-trips the new kinds + `IsMindMapBranch` → SVG/PNG
export shows the tapered branches.
